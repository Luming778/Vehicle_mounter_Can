# Vehicle Control System - STM32F103 Multi-Node
基于三片STM32F103的车载多节点智能控制系统，支持CAN总线通信、语音控制与WiFi远程控制。

---

## 项目简介
本项目采用**分布式三节点架构**，使用三片STM32F103单片机实现车载设备的本地控制、状态显示、语音控制与远程控制。
- **主控屏节点**：负责设备状态显示、语音指令解析、CAN总线通信与WiFi服务器交互
- **设备节点一**：控制车灯、车窗、天窗模块
- **设备节点二**：控制车门模块

三个节点通过CAN总线实现稳定通信，设备状态实时同步至主控屏与服务器，支持本地、语音、远程三种控制方式。

---

## 硬件组成
- 主控芯片：3片 STM32F103 系列单片机
- 通信方式：CAN总线、ESP8266 WiFi模块
- 外设模块：语音识别模块、显示屏
- 被控设备：车灯、车门、车窗、天窗

---

## 节点功能说明

### Vehicle_screen_controller（主控屏节点）
1. 实时显示设备一、设备二的车灯、车门、车窗、天窗状态
2. 接收语音模块指令，解析后通过CAN总线向设备节点下发控制命令
3. 通过WiFi模块将设备状态定时上传至服务器
4. 接收服务器下发的控制指令，通过CAN总线转发至对应设备节点

### Vehicle_module_controller_1（设备节点一）
1. 负责控制：车灯、车窗、天窗
2. 本地可通过按键独立控制模块状态
3. 状态变化后通过CAN总线主动上报主控屏
4. 执行主控屏下发的CAN控制指令

### Vehicle_module_controller_2（设备节点二）
1. 负责控制：车门
2. 本地可通过按键独立控制车门状态
3. 状态变化后通过CAN总线主动上报主控屏
4. 执行主控屏下发的CAN控制指令

---

## 核心功能
1. **状态实时显示**
主控屏实时刷新显示车灯、车门、车窗、天窗的当前工作状态。

2. **设备本地控制**
设备节点一、二可通过按键独立控制对应模块，状态自动上报主控屏。

3. **语音智能控制**
语音模块识别控制指令后，由主控屏通过CAN总线向设备节点下发命令，实现语音控制。

4. **WiFi远程控制**
主控屏通过WiFi模块将设备状态上传至服务器，同时服务器可下发指令，远程控制车载设备状态。

5. **CAN总线通信**
三个节点通过CAN总线完成状态上报与指令下发，通信稳定可靠，满足车载场景需求。

## 使用说明
1. 将三个工程分别下载到对应的STM32F103单片机中
2. 正确连接CAN总线、WiFi模块、语音模块与显示屏
3. 上电后系统自动初始化并运行
4. 支持本地按键控制、语音指令控制、服务器远程控制三种控制方式
